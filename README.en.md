# Synthetic Users Lab

> Simulate synthetic user personas against any product flow — get UX scores, friction points, and actionable issues without recruiting a single real user.

🇪🇸 [Versión en español](README.md)

---

## What is this?

**Synthetic Users Lab** is an AI-powered UX testing tool that lets you run simulated user research in seconds. Select from a library of user personas (or create custom ones), point them at a URL or product description, and get back a full simulation: step-by-step journey, UX issues, fit scores, and verbatim reactions — all generated by Gemini 2.5 Flash.

Built as a side project to explore AI-assisted product research. Open source and free to self-host.

---

## Who is it for?

- **Product designers** validating onboarding flows and pricing pages before user testing
- **Founders & PMs** getting fast qualitative signal on new features or landing pages

## How it works

The app runs a 4-step wizard:

**Step 1 — Pick personas**
Choose from 12+ preset profiles (Early Adopter, Busy Manager, Accessibility User, UX Researcher, etc.) or create a custom one. Each persona has tech level, frustration threshold, and behavioral traits.

**Step 2 — Define the flow**
Enter a URL, GitHub repo, or plain text description of the product/flow to evaluate. Paste in product context or auto-fill it from URL metadata.

**Step 3 — Simulation runs**
For each selected persona, the backend runs a two-phase pipeline:
1. **Objective analysis** — Gemini maps the UI elements, navigation flow, and objective UX issues
2. **Persona simulation** — Gemini re-evaluates the flow through the persona's lens, producing a step-by-step journey, issues, fit score, and verbatim reaction

**Step 4 — Results**
Aggregated metrics (avg UX score, total issues, critical count, retention), per-persona breakdowns with issue filtering by category (UX / UI / Product / Copy), and export to Markdown.

## Setup

```bash
# 1. Clone and install
git clone https://github.com/marcomarinodesign/synthetic-users-lab.git
cd synthetic-users-lab
npm install

# 2. Configure environment
cp .env.example .env.local
# Add your Gemini API key (free tier works):
# GEMINI_API_KEY=your_key_here
# Get one at: https://aistudio.google.com/app/apikey

# 3. Run locally
npm run dev        # Frontend at localhost:3000 + API at localhost:3001

# Other commands
npm run build      # Production build
npm run lint       # ESLint
npm run test       # Unit tests
```

## Key features

- **12+ preset personas** — regional, industry, accessibility, and expert profiles (UX Researcher, UI Designer, Product Strategist)
- **Two-phase AI pipeline** — objective structural analysis anchors persona simulations, reducing hallucinations
- **URL auto-enrichment** — fetches and extracts product context from any URL automatically
- **Expert lens mode** — specialist personas give professional-grade UX audits with implementable tickets
- **Simulation history** — past runs saved in localStorage; restore any result without re-calling the API
- **Baseline comparison** — compare current results against any past simulation with delta metrics (+/-)
- **Export to Markdown** — copy or download full results for async sharing or tickets
- **Privacy-first analytics** — optional Plausible integration, no cookies, no consent banners

## Project structure

```
api/
├── simulate.ts              # Vercel endpoint: runs simulation pipeline
├── simulate-stream.ts       # Vercel endpoint: SSE streaming version
├── fetch-content.ts         # Vercel endpoint: fetches and extracts URL text
├── fetch-url-meta.ts        # Vercel endpoint: extracts metadata for auto-fill
└── simulation-core.js       # Shared logic: prompts, JSON repair, Gemini client

src/
├── App.tsx                  # 4-step wizard + results view
├── main.tsx                 # React entry point
├── components/
│   ├── ds/                  # Domain components (PersonaCard, ResultCard, MetricCard…)
│   └── ui/                  # UI primitives (Button, Dialog, Tabs…)
├── lib/
│   ├── analytics.ts         # Plausible event helpers
│   ├── storage.ts           # localStorage history (save / load / delete)
│   ├── simulation-service.ts# Batch runner + input preparation
│   └── personas.ts          # Preset persona definitions
├── hooks/
│   └── useRunSimulation.ts  # Async simulation orchestration with phase tracking
└── types/                   # Domain types (Persona, SimulationResult, Issue…)
```
